我是怎样把vscode打造成趁手的markdown+latex工具的

前情提要 插件需求

首先介绍下我用markdown和latex写博客的时候经常遇到的问题：

  1. 因为需要中英文混合输入，经常避免不了的问题就是中文的标点它解析不了，，，
  2. latex需要敲好多重复的内容，敲的时候很累，而且需要修改的时候也很麻烦
  3. latex有时候会忘记上标和下标的括号
  4. 图片插入不方便

目前没解决的问题是,在文件内部实时预览latex并折叠latex代码,和拖拽实现图片自动上传图床和插入.

目前的想法：

  1. 用macro解决mathematica code,可能正则表达式会挺难写
  2. 怎么在snippet上链接sympy来实时运算

解决过程

所以呢，我们就做了这些配置：

主要就是利用 vscode 自带的 snippet 和 keybinding 功能把我需要的中文标点问题和快捷输入latex问题解决了. 在 vscode 环境中我完全使用英文标点,因为等距的标点也很清晰,不会字堆在一起, 同时使用 keybinding 不会影响外部的输入法.

首先是插件：

我使用了markdown all in one这款插件，支持latex的语法补全，表格对齐和todo的快捷操作等，强烈推荐

之后是user snippet的配置：

更多信息请看snippet 的文档, 这里的信息比较全, 根据文档可以自己定制需要的配置.

{   //目的是把中文标点映射成英文的符号
    //这里是中文符号映射需要的snippet,待会需要调用这里的snippet做keymap 
	"Bracket left": {"prefix": "[","body": "[$1]"},
	"Bracket right":{"prefix": "]","body": "]"},
	"Shift bracket left":{"prefix": "{","body":"{$1}"},
	"Shift bracket right":{"prefix": "}","body": "}"},
	"Smaller":{"prefix": "<","body": "<"},
	"Greater":{"prefix": ">","body": ">"},
	"Backquote":{"prefix": "`","body": "`"},
	"Backslash":{"prefix": "\\","body": "\\"},
	"Dollar":{"prefix": "$","body": "$"},
	"Underline":{"prefix": "_","body": "_"},
	"Period":{"prefix": ".","body": "."},
	"Comma":{"prefix": ",","body": ","},
	"Parenthesis left":{"prefix": "(","body": "($1)"},
	"Parenthesis right":{"prefix": ")","body": ")"},
}

{
    //这里是markdown要用的snippet
    //包括我每天要写的worklog和一些latex要用的快捷键
    "worklog":{
    "prefix": "worklog",
    "body": [
        "# $CURRENT_MONTH_NAME_SHORT $CURRENT_DATE","",
        "## 计划：","",
        "- [ ] $2","",
        "## 计划外","",
        "$3","",
        "## 日志","",
        "$4",
    ],
    "description": "output my working log"
	},
	"beginmatrix":{
		"prefix":"bem3",
		"body": [
			"\\begin{bmatrix}",
			"    $1 & $2 & $3 \\\\\\\\ $4 & $5 & $6 \\\\\\\\ $7 & $8 & $9",
			"\\end{bmatrix}"
		],
		"description": "construction matrix"
    },
    
    //这两个snippet会比较常用,因为latex需要经常输入上标和下标,这两个同时也被keymap了

	"add bracket _":{
		"prefix": "_",
		"body": "_{$0}"
	},
	"add bracket ^":{
		"prefix": "^",
		"body": "^{$0}"
	}
}

做完这一步之后我们还需要在markdown里面打开代码提示：

//打开markdown代码提示
    "[markdown]": {
        "editor.quickSuggestions": {
            "other": true,
            "comments": false,
            "strings": false
        }
    }
